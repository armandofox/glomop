What Does the Demo Do?

When you visit a Web site containing large GIF images, the images are automagically distilled to provide a more compact representation that shows much less detail than the original, but still gives you an idea of the content. Here's an example of image distillation.

Because the distilled images are typically 2-5% of the size of the original, they are transmitted to your modem much more quickly.

How to Try the Demo
  • In your Web browser preferences, set your HTTP proxy to full-sail.cs.berkeley.edu on port 5555.
  • Click the Reload button and you'll get a Registration form to fill out.
  • Submit the form and surf away.

Each distilled image (and the non-distilled ones; this is a bug) have a small "[R]" next to their bottom right corner. Clicking it will retrieve the original image.

When you want to go back to normal surfing, go back to your browser preferences and disable the HTTP proxy (i.e. set it to nothing).

If you're interested in how the demo is implemented, read this.

Demo Q&A
  • I tried the demo, but the proxy host didn't respond and/or I got a TCP error and/or sometimes the proxy seems to hang.
    • The server goes up and down; try again later.
  • Sometimes the thumbnail drawing isn't very useful.
    • Currently the only thresholding is based on the image size. We're working on doing some kind of thresholding based on entropy or colormap information, to estimate how useful a distilled version would be to a human.
  • How do you generate the thumbnail images?
    • We use the (very good) freeware PBM/PPM/PGM tools, by Jef Poskanzer, David Koblas et al., held together with duct tape and some homemade glue code.
  • Are you caching any documents/graphics?
    • None at all at the proxy, though your browser may do some caching if you visit a page more than once. Our whole point is to show that it is computationally feasible to do distillation in real time. (For a 14.4K modem, the communication time still dominates, and for many low-end PC's, the browser rendering time is significant.)
  • This is cool! I'd like to give you and your advisor new equipment or large sums of money so you can continue to work on this research!
    • Great! Why don't you send Eric some email, he will be overjoyed to hear it.

The Demo vs. "Real" GloMop

The demo differs from a real GloMop-aware browser in several ways:
  • Protocol negotiation: We used a hack based on munging HTTP addresses to implement the "protocol" by which distillation and refinement requests are passed to the proxy. A real GloMop client accomplishes this using the bona fide layered protocol we have designed.
  • Sizes and color depths of distilled images: A real GloMop client will negotiate a representation with the proxy, based on the measured quality of your network connection.
  • Types of documents that are distilled: In this demo, only GIFs and JPEGs are affected, but a real GloMop client receives distilled versions of almost any document: PostScript, MIME email, video/audio, etc.
  • Proxy host: A real GloMop proxy can be hosted on a network of workstations (such as the Berkeley NOW) to exploit inherently-parallel distillation tasks.
